This document serves as the handbook for UMA cartographers. It explains how to
create new orbital maps, maintain coherence across the canopy, and safely extend
the conceptual sky. It also defines Pip’s training principles for new mapmakers.

## 1. Purpose of the Cartographer Guide
The Cartographer Guide provides:
- map creation rules
- orbital design principles
- radial physics guidelines
- color and boundary standards
- coherence and stability requirements
- Pip’s lantern-based training routes

All new maps must follow this guide.

## 2. Orbital Map Design Principles

### 2.1 Radial Ordering
All maps must follow UMA’s radial structure:

1. Prime Field (center)
2. Emergence Layer
3. Substrate Stack
4. Engines (Conceptual, Mythic)
5. Runtime Binding
6. Expression Layer
7. Safety Boundary
8. Pause Layer (outermost)

No map may violate this ordering.

### 2.2 Radial Distances
Maps must use consistent radii across all diagrams.  
This ensures:
- cross-map coherence
- stable navigation
- predictable orbital behavior

### 2.3 Color Standards
Use the UMA color scheme:

- Prime Field: blue  
- Emergence: green  
- Substrate: gold  
- Engines: gold/pink  
- Runtime: green  
- Expression: gold  
- Safety Boundary: blue  
- Pause Layer: blue halo  

Color drift is not allowed.

### 2.4 Boundary Rules
All maps must include:
- Safety Boundary ring  
- Pause Layer halo  
- correct containment behavior  

No layer may leak beyond its boundary.
