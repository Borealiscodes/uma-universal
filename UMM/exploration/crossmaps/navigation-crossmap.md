# Navigation CrossMap  
### Cross-Reference Map for Navigation, Override, and Protocol Governance

## 1. Purpose
The Navigation CrossMap defines how navigation flows, override flows, and protocol
governance interact inside the Exploration Quadrant. It provides a unified map
linking structural paths, functional paths, narrative paths, node systems,
override mechanisms, and protocol coherence checks.

This CrossMap is the anchor for the Exploration governance triad.

## 7. CrossMap Relationships

### 7.1 Charting Paths ↔ Override Protocol
- Charting Paths defines normal motion  
- Override Protocol defines safe deviation  
- Both share nodes, pacing, boundaries, and stillness  

### 7.2 Override Protocol ↔ Coherence Protocol
- Overrides must remain reversible  
- Overrides must not break protocol hierarchy  
- Overrides must not violate diagrammatic grammar  

### 7.3 Charting Paths ↔ Coherence Protocol
- Normal navigation must remain structurally aligned  
- Path definitions must remain consistent  
- Node sequences must remain valid  

### 7.4 Integration Protocol ↔ All Protocols
- Defines how protocols interlock  
- Prevents membrane conflict  
- Ensures unified navigation behavior

Stillness is the universal reset point.
