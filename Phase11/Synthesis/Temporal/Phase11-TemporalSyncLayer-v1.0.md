# **III. Temporal Modes**

Each sync mode supports multiple strategies:

### **FrameSyncMode**
```
LOCKSTEP        // all runtimes advance together
ADAPTIVE        // runtimes adjust to a global cadence
FREEFLOW        // runtimes free-run but align at checkpoints
```

### **HapticSyncMode**
```
PHASE_LOCK      // haptics locked to globalTimecode
SEMANTIC_LOCK   // haptics locked to meaning transitions
DEVICE_NATIVE   // device-specific timing with NDH correction
```

### **AudioSyncMode**
```
BEAT_LOCK       // audio aligned to rhythmic structure
EVENT_LOCK      // audio aligned to NDH events
SPATIAL_LOCK    // audio aligned to XR spatial transforms
```

### **XRPoseSyncMode**
```
POSE_LOCK       // XR transforms locked to globalTimecode
DRIFT_CORRECT   // XR transforms corrected periodically
HOLONOMY_LOCK   // XR transforms aligned to η geometry
```

### **OperatorSyncMode**
```
OPERATOR_PHASE  // operators run in temporal phases
OPERATOR_EVENT  // operators run on NDH events
OPERATOR_CASCADE// operators cascade with timing offsets
```
