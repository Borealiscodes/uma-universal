# 🌌 **Phase‑11 Global Sensory Envelope (GSE) v1.1 — Expanded**

The GSE is the **single NDH sensory object** that can be projected into:

Unity → Unreal → OpenXR → VisionOS → Android → iOS → DualSense → SteamVR → WebXR.

It is the “one object → all runtimes” payload.

---

# **I. Core NDH Fields (Phase‑10 Invariants)**

These are the invariant sensory primitives:

```
chi                // cue amplitude
nu                 // volatility
eta                // holonomy
phi[3]             // adjacency field (x,y,z)
basin              // Calm, Wonder, Curiosity, Reflection
```

These remain unchanged — they are the backbone of NDH.

---

# **II. Phase‑11 Expansion Fields**

Phase‑11 adds **cross‑system synthesis fields** that allow the GSE to be projected into multiple runtimes simultaneously.

---

## ⭐ **1. Runtime Projection Envelope**

These fields describe how the GSE should be interpreted by each runtime:

```
runtimeProfile {
    unityProfile
    unrealProfile
    openxrProfile
    visionosProfile
    androidProfile
    iosProfile
    dualsenseProfile
    steamvrProfile
    webxrProfile
}
```

Each profile contains:

- amplitude scaling  
- spatial transform rules  
- haptic meaning mapping  
- accessibility operator mapping  
- stability negotiation parameters  

This is the “broadcast layer.”

---

## ⭐ **2. Unified Haptic Meaning Model**

This is the Phase‑11 innovation:  
a **cross‑device haptic meaning** field.

```
hapticMeaning {
    semanticIntensity
    semanticSharpness
    semanticTexture
    semanticTemporalShape
}
```

These are *not* raw haptic parameters.  
They are **meaning parameters** that each device translates into its own hardware‑specific form.

This is how Taptic Engine, Android, DualSense, XR controllers, and WebXR haptics all express the **same sensation**.

---
