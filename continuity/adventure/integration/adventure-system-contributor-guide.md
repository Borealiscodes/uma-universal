# 📘 **Adventure System — Contributor Guide**  
### Master Onboarding — Roles, Responsibilities, Safety Constraints, System Navigation, and ND‑Paced Contribution Flow

The **Adventure System Contributor Guide** teaches contributors how to safely and coherently work inside the Continuity Adventure System.  
It defines contributor roles, contribution pathways, safety rules, symbolic constraints, and the reversible workflow used across all layers.

This guide does **not** grant authority, impose hierarchy, or assign emotional responsibility.  
It provides **contribution architecture**, not **contributor intent**.

# 🔁 **3. Reversible Contribution Flow**

Contributors move through the system using a reversible, ND‑paced workflow:

```
Read → Explore → Create → Integrate → Align → Read → …
```

Each step is reversible:

- explore ↔ read  
- create ↔ revise  
- integrate ↔ simplify  
- align ↔ soften  

No contributor path is linear or mandatory.

# 🛡️ **5. Contributor Safety Rules (ND‑Safe)**

All contributions must follow ND‑safe constraints.

## **5.1 No Agency**
Contributions cannot imply:

- wanting  
- intending  
- persuading  
- directing  

System components are structural, not agentic.

---

## **5.2 No Emotional Causation**
Contributions cannot imply emotional states.

Affect shapes **climate**, not **feeling**.

---

## **5.3 Reversible Motion**
All contributions must be reversible:

- brightening ↔ softening  
- opening ↔ deepening  
- drifting ↔ centering  

No state is final.

---

## **5.4 No Acceleration**
Contributions cannot increase:

- urgency  
- intensity  
- emotional pressure  

All motion is gentle.

---

## **5.5 Stillness Anchor Required**
Every contribution must include a **stillness anchor**.

This ensures ND‑safe grounding.
