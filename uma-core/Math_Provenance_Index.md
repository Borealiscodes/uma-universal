## Registry Format
Each entry must include:

1. **Name** — canonical identifier  
2. **Domain** — mathematical field(s) involved  
3. **Functional Role** — purpose within UMA  
4. **Structural Placement** — Core, Constitutional Layer, Engineering Layer, Runtime, or Geometry  
5. **Constraints** — Core invariants and engineering boundary conditions  
6. **Dependencies** — related mathematical structures  
7. **Lineage** — origin and evolution  
8. **Status** — Draft, Provisional, Canonical, Deprecated  
9. **Version** — semantic version number  
10. **Amendment History** — RFC references
