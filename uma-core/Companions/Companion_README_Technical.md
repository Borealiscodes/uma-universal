## 1. Scope  
This document provides the **technical specification** for the UMA Companion Constellation.  
Companions are **non‑constitutional**, **non‑governance**, **human‑side modules** designed to support:

- cognitive safety  
- decompression  
- reflection  
- narrative exploration  
- academic reasoning  

All Companion modules operate within the **Philosophy Couch Interpretation Space**, which functions as the unified interpretive layer for human‑side cognitive operations.

---

## 2. System Architecture Overview  
The Companion Constellation is structured as a **layered cognitive system**:

### Layer 0 — Interpretation Layer  
- **Philosophy Couch Interpretation Space**  
- Provides boundary conditions, pacing constraints, and IC/OC separation.

### Layer 1 — Comfort Layer  
- **Decompression Companion**  
- Provides cognitive downshifting and load reduction.

### Layer 2 — Integration Layer  
- **Reflection Companion**  
- Provides meaning‑making, conceptual integration, and cognitive stabilization.

### Layer 3 — Exploration Layer  
- **Play Mode Universal**  
- Provides narrative simulation, IC/OC roleplay, and conceptual prototyping.

### Layer 4 — Analytical Layer  
- **Academic & Research Dice Mode**  
- Provides structured academic reasoning using D²iCE perspective faces.

Each layer is modular, reversible, and non‑intrusive.

## 4. Inter‑Module Relationships

### 4.1 Mandatory Dependencies  
- All modules depend on the Philosophy Couch Interpretation Space.  
- Reflection depends on Decompression.  
- Play Mode depends on Reflection and Decompression.  
- Academic Dice Mode depends on Reflection and Decompression.

### 4.2 Optional Dependencies  
- Play Mode may optionally invoke Academic Dice Mode for structured world logic.  
- Academic Dice Mode may optionally invoke Play Mode for narrative‑based analysis.

---

## 5. Safety Constraints

### 5.1 Cognitive Safety  
- All modules must maintain ND‑safe pacing.  
- No module may induce recursive cognition.  
- All transitions must be reversible.

### 5.2 Emotional Safety  
- Decompression and Reflection must provide non‑intrusive support.  
- Play Mode must maintain IC/OC separation.  
- Academic Dice Mode must prevent bleed into personal reasoning.

### 5.3 Architectural Safety  
- No Companion may interact with UMA’s constitutional architecture.  
- No Companion may modify governance, lineage, or protocol layers.

---

## 6. Canonical Statement  
The Companion Constellation is a **human‑side cognitive system**.  
It provides comfort, reflection, exploration, and academic reasoning.  
It does **not** interact with UMA’s constitutional architecture.
