## Purpose

# 2. Correction Interface Layer
The Correction Interface applies repairs safely, without destabilizing UMA.

## 2.1 Correction Categories
Corrections are classified as:
- Constitutional  
- Mathematical  
- Geometric  
- Epistemic  
- Provenance  
- Cross‑Layer  

## 2.2 Correction Mechanisms
Corrections use:
- Pause Layer Correction Protocol™  
- Math Check Protocol™  
- CME‑Crossmap™  
- Provenance Correction Log™  
- Roots lineage updates  

## 2.3 Correction Flow
1. **Freeze** — halt forward motion  
2. **Identify** — classify correction  
3. **Align** — apply constitutional + cross‑layer constraints  
4. **Record** — update Roots + logs  
5. **Release** — resume development  

## 2.4 Correction Safety Rules
- No correction may override the Core  
- No correction may introduce symmetry  
- No correction may bypass provenance  
- No correction may alter historical truth  
- No correction may skip diagnostic review  
- No correction may proceed without Pause Protocol activation
