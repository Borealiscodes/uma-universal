# **3. Traversal Plane Isolation Rules (v1.2)**

These rules define what **TRV‑CHS‑OL** may interact with, must avoid, and must be supervised by.

---

## **Rule 1 — Cognitive Isolation (SYS‑CHS → TRV‑CHS‑OL)**  
**Traversal may read SYS‑CHS cognitive nodes but may not modify them.**

Prohibited:

- node creation  
- node deletion  
- node merging  
- adjacency modification  

Explore: **SYS‑CHS**

---

## **Rule 2 — Traversal Path Isolation (TRV‑CHS‑OL)**  
**Traversal paths must remain stable, SIAP‑approved, and drift‑free.**

Requirements:

- no traversal‑driven drift  
- no traversal‑driven propagation vectors  
- no traversal‑driven subsystem expansion  
- no traversal‑driven adjacency corruption  

Explore: **TRV‑CHS‑OL**
