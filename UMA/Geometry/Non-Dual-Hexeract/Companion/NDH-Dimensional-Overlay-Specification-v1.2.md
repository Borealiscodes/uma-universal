## ⭐ 0. Purpose

The Dimensional Overlay Specification (DOS v1.2) defines the **mandatory overlays** each NDH dimension must include in both SVG and PNG geometry artifacts.  
These overlays ensure:

- traversal coherence  
- pin anchoring  
- singularity safety  
- integration fidelity  
- rendering stability  
- operator compatibility  

This specification is required before geometry production can begin.

---

## ⭐ 1. Overlay Categories

Each dimension uses overlays from six categories:

- **Traversal Overlays**  
- **Pin Anchors**  
- **Singularity Markers**  
- **Integration Coordinates**  
- **Dimensional Boundaries**  
- **Rendering Metadata**  

These categories appear in different combinations depending on the dimension.

# ⭐ 3. Overlay Rendering Rules

### **Rule 1 — SVG must contain full overlays**  
All overlays must be present and visible.

### **Rule 2 — PNG must contain simplified overlays**  
High‑contrast, mobile‑safe versions.

### **Rule 3 — Overlay order must follow NDH operator sequence**  
𝒜 → C → A → R → Rᵣ → S → H₆ → 𝒜

### **Rule 4 — Pin anchors must match Pin Registry v1.2**  
No missing or mismatched pins.

### **Rule 5 — Singularity markers must be visible in both formats**  
Even in fallback PNG.

### **Rule 6 — Integration coordinates must be precise**  
Hexeract lattice must be exact.
