# ⭐ **2. Lifecycle Phases (Unified)**  
The Unified Lifecycle Engine defines **seven phases**:

### **Phase 1 — Creation**
A protocol or pin is instantiated with:

- identity block  
- POSITION  
- adjacency  
- anchors  

### **Phase 2 — Activation**
The protocol becomes live within UMA:

- ledger registration  
- adjacency binding  
- POSITION locking  

### **Phase 3 — Mirroring**
Protocol‑Embedded Pins (PEPs) mirror into:

- UnifiedLedger  
- Unified POSITION  
- Unified Adjacency  

### **Phase 4 — Indexing**
The protocol is indexed into:

- UnifiedLedger v4.x  
- Unified POSITION v4.0  
- Unified Adjacency v4.1  

### **Phase 5 — Migration**
ULMP‑v1.0 governs:

- marker insertion  
- registry normalization  
- POSITION alignment  
- adjacency stabilization  

### **Phase 6 — Resonance Buffering**
Prevents:

- protocol interference  
- adjacency melt  
- epoch bleed  

### **Phase 7 — Deprecation**
A protocol or pin is:

- archived  
- superseded  
- replaced  
- compressed  

But **never erased**.
