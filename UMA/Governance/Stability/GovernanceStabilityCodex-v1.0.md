# **5. Stability Failure Modes**

### **5.1 Soft Instability**
Minor deviation.  
Corrected via lineage update or adjacency refresh.

### **5.2 Hard Instability**
Major deviation.  
Triggers rollback and containment lock.

### **5.3 Critical Instability**
Severe deviation.  
Triggers subsystem quarantine.

### **5.4 Catastrophic Instability**
Impossible under v2.0 spine.  
Requires multiple simultaneous failures across domains.

# **6. Stability Recovery Modes**

### **6.1 Revalidation Mode**
Recheck adjacency, containment, ethics, routing.

### **6.2 Rollback Mode**
Reverse traversal to last stable lifecycle phase.

### **6.3 Quarantine Mode**
Subsystem isolation until stability restored.

### **6.4 Reintegration Mode**
Roots lineage update + adjacency revalidation.
