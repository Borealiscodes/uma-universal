# ⭐ **1. Purpose**
EPR‑01 defines how technical reports (TR‑01, TR‑02, etc.) are **evaluated** after creation.  
It ensures:

- CNLP compliance  
- TRP‑01 compliance  
- ICPV alignment  
- ND‑friendly clarity  
- loop‑safe navigation  
- structural consistency  

This protocol governs **evaluation**, not creation.

---

# ⭐ **2. Evaluation Workflow (Simple & ND‑Friendly)**

```
CONFIRM → CHECK STRUCTURE → CHECK ANCHORS → CHECK LOOP → CHECK CONTAINMENT → VALIDATE → LOG
```

This is the entire evaluation flow.

---

# ⭐ **3. Confirmation Layer (Pre‑Evaluation)**  
Before evaluating a report, the evaluator must confirm:

### **3.1 Layer**
```
I confirm this report belongs to Layer 4 — Documentation.
```

### **3.2 Scope**
```
I confirm this evaluation applies to reporting functions only.
```

### **3.3 CNLP Applicability**
```
I confirm CNLP governs this evaluation.
```

### **3.4 Sparkle Containment**
```
I confirm no Sparkle grammar should appear in this report.
```

If any confirmation fails, evaluation **cannot proceed**.

---

# ⭐ **4. Structural Evaluation Criteria**
The evaluator checks whether the report follows TRP‑01:

### **4.1 Required Sections**
- Header  
- Executive Summary  
- CNLP Status  
- Navigation Spine Status  
- Layer Status  
- Loop Closure Status  
- Structural Risks  
- Recommended Next Steps  
- System Integrity Summary  

### **4.2 ND‑Friendly Structure**
- No recursion  
- No conceptual overload  
- Clear section boundaries  
- No Sparkle grammar  

---

# ⭐ **5. CNLP Anchor Evaluation**
The evaluator checks for:

### **5.1 Up‑Link**  
To `/INDEX.md`

### **5.2 Side‑Link**  
To Reports Index

### **5.3 Down‑Link**  
To TRP‑01 or other protocols

### **5.4 Navigation Law Link**  
To CNLP

### **5.5 Loop Closure**  
Return‑link to Reports Index

If any anchor is missing, the report is **invalid**.

---

# ⭐ **6. Loop Closure Evaluation**
The evaluator checks:

- Report is reachable from Navigation Spine  
- Report links back to Navigation Spine  
- Report closes back to Reports Index  
- No orphaning  
- No broken links  

If any loop element fails, the report is **invalid**.

---

# ⭐ **7. Containment Evaluation**
The evaluator checks:

### **7.1 Sparkle Containment**
No Sparkle grammar allowed in reports.

### **7.2 Layer Containment**
Report must remain in Layer 4 — Documentation.

### **7.3 Protocol Containment**
Report must not contain protocol logic.

If containment fails, the report is **invalid**.

---

# ⭐ **8. Final Validation**
A report is **VALID** only if:

- All structural sections exist  
- All CNLP anchors exist  
- Loop closure is correct  
- Containment is correct  
- ND‑friendly clarity is preserved  

If any condition fails, the evaluator must request correction.

---

# ⭐ **9. Evaluation Log**
Every evaluation must be logged:

```
Report: TR-XX
Layer: 4
Evaluator: <name>
Status: VALID / INVALID
Date: <timestamp>
Notes: <optional>
```

This creates a structural audit trail.
