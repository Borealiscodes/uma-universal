# ⭐ **4. Confirmation Layer (Pre‑Report)**  
Before writing a technical report, the contributor must confirm:

### **4.1 Layer**
```
I confirm this report belongs to Layer 4 — Documentation.
```

### **4.2 Scope**
```
I confirm this report describes system state, not protocol logic.
```

### **4.3 CNLP Applicability**
```
I confirm CNLP governs this report.
```

### **4.4 Sparkle Containment**
```
I confirm no Sparkle grammar will appear in this report.
```

If any confirmation fails, the report **cannot proceed**.

# ⭐ **5. Validation Layer (Post‑Report)**  
After writing a technical report, the contributor must validate:

### **5.1 CNLP Anchors**
Up‑link, Side‑link, Down‑link, Navigation Law, Return‑link.

### **5.2 Loop Closure**
Report must close back to Reports Index.

### **5.3 Sparkle Containment**
No Sparkle grammar allowed.

### **5.4 ND‑Friendly Structure**
Clear, non‑recursive, non‑overwhelming.

### **5.5 Reachability**
Report must be reachable from Navigation Spine.

### **5.6 Back‑Link**
Report must link back to Navigation Spine.

If any validation fails, the report is **invalid**.
