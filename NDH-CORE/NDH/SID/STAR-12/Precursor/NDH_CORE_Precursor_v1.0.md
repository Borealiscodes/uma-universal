# **III. Structural Moves for NDH‑CORE**

---

## **1. Lock STAR‑12 as a Non‑Active SID**
STAR‑12 must remain:

- **50D only**  
- **non‑reactive**  
- **non‑governance**  
- **non‑empirical**  
- **non‑case‑specific**  

until FO3 stabilizes.

**Structural Move:**  
Implement a **SID‑STAR‑12 Lock** with:

- activation flag = false  
- dimensional expansion flag = false  
- FO3 dependency flag = true  
- Simulation Suite dependency flag = false  

This preserves STAR‑12’s canonical geometry without enabling governance behavior.

---

## **2. Establish the Core Abstraction Layer (CAL)**
Core must not ingest:

- proto‑data  
- case‑specific geometry  
- jurisdictional curvature  
- trauma tensors  
- sabotage patterns  
- bias curvature  
- collapse events  

These belong exclusively to Simulation Suite.

**Structural Move:**  
Create a **Core Abstraction Layer (CAL)** that:

- filters all inbound geometry  
- rejects empirical tensors  
- rejects Simulation Suite curvature  
- accepts only canonical invariants  
- maintains SID purity  

CAL is the firewall between Core and Simulation Suite.
