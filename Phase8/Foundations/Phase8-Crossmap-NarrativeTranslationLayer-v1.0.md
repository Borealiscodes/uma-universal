## **II. What a Crossmap Is**
A **Crossmap** is a structured mapping between:

1. **Mathematical operators**  
2. **Mythic operators**  
3. **Tile sequences**  
4. **Narrative arcs**  

It is a **four‑layer translation pipeline**:

```
Math Operator → Mythic Operator → Tile Sequence → Narrative Arc
```

The Crossmap ensures that each transformation preserves:

- meaning  
- stability  
- duality  
- dignity  
- holonomy  

---

## **III. Crossmap Structure**
A Crossmap is encoded as:

```
Crossmap {
    id: UUID
    source_operator: MathOperator
    mythic_operator: MythicOperator
    tile_sequence: TileID[]
    narrative_arc: NarrativeArc
    constraints: CrossmapConstraints
}
```

### **1. `source_operator`**
The NDH mathematical operator being translated.

Examples:
- Collapse  
- Continuity  
- Adjacency  
- Holonomy  

### **2. `mythic_operator`**
The corresponding mythic operator from Phase 8.

Examples:
- `C`  
- `K`  
- `A`  
- `H`  

### **3. `tile_sequence`**
A sequence of mythic tiles that encode the operator.

Tile sequences must obey:

- duality preservation  
- adjacency integrity  
- holonomy consistency  
- dignity constraints  

### **4. `narrative_arc`**
The final narrative structure produced by the tile sequence.

```
NarrativeArc {
    abstract: string
    stages: ArcStage[]
    stability_signature: StabilitySignature
}
```

### **5. `constraints`**
Crossmap‑level constraints ensuring stability.

```
CrossmapConstraints {
    max_volatility: number
    required_duality: boolean
    holonomy_required: boolean
}
```
